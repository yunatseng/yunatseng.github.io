2. flexbox 介紹

  1. 最外層 display:flex / 選擇方向 flex-direction: row | row-reverse | column | column-reverse;
  2. 內層 決定伸縮性 flex:1|0 / 決定裡頭物件位置 display:flex 水平方向 justify-content: center; 垂直方向 align-items: center;
  3. 內層元素性質詳細解釋
  • flex-grow: 元件的伸展性，是一個數值，當空間分配還有剩餘時的當前元件的伸展性，預設值為 0，如果設置為 0 則不會縮放。
  • flex-shrink: 元件的收縮性: 元件的伸展性，是一個數值，當空間分配還不足時的當前元件的收縮性，預設值為 0，如果設置為 0 則不會縮放。
  • flex-basis: 决定 flex 元素的内容盒（content-box）尺寸。當一個元素同時被設置了 flex-basis 和 width（或者在 flex-direction：column 情況下設置了高度）時，flex-basis 具有更高的優先級。

5. z-index 何時失效

  1. z-index 只對定位元素有效，也就是設置了 position 屬性的元素，position 的屬性值如下：absolute-絕對定位、relative-相對定位、fixed-固定定位、inherit-繼承父元素定位，static-靜態定位。這裡要注意，並不是所有的定位設置都有效果，absolute、relative 和 fixed 是肯定有效果的，inherit 取決於父元素，如果父元素沒有設置定位則 z-index 無效，注意低版本IE瀏覽器不支持這個值。最後 static 這個靜態定位，其實是默認值，表示當前元素不進行定位，所以如果元素設置了這個屬性值，其實是和沒有設置是一樣的，會使元素忽略掉 z-index 屬性，使其不起作用。

  2. z-index 只能是整數（恩..）

JavaScript 部分

1. 解釋同步和非同步

JS 是單執行緒的語言。

單執行緒就意味著，所有任務需要排隊，前一個任務結束，才會執行後一個任務。如果前一個任務耗時很長，後一個任務就不得不一直等著。
HTML5 提出 Web Worker 標準，允許 JavaScript 建立多個執行緒，但是子執行緒完全受主執行緒控制，且不得操作 DOM。所以，這個新標準並沒有改變 JavaScript 單執行緒的本質。
為了解決這個問題，Javascript 語言將任務的執行模式分成兩種：同步（Synchronous）和非同步（Asynchronous）。於是，所有任務可以分成兩種，一種是同步任務（synchronous），另一種是非同步任務（asynchronous）。
同步任務指的是，在主執行緒上排隊執行的任務，只有前一個任務執行完畢，才能執行後一個任務；非同步任務指的是，不進入主執行緒、而進入”任務佇列”（task queue）的任務，只有”任務佇列”通知主執行緒，某個非同步任務可以執行了，該任務才會進入主執行緒執行。

3. var、let、const 的差別

  1. 可以重新宣告、可以重新賦值 => var （function scope）
  2. 不能重新宣告、可以重新賦值 => let（block scope）
  3. 不能重新宣告、不能重新賦值 => const（block scope）
  4. const 對物件的屬性可以重新賦值（mutatable）

4. 請解釋和手寫 Promise

Promise 是一個表示非同步運算的最終完成或失敗的物件。
一個 Promise 物件處於以下幾種狀態：

  • 擱置（pending）：初始狀態，不是 fulfilled 與 rejected。
  • 實現（fulfilled）：表示操作成功地完成。
  • 拒絕（rejected）：表示操作失敗了。

Vue 部分

1. 為什麼要使用 Vue Directive

為了實現 View 和 ViewModel 的分離，我們必須封裝 DOM 操作，使用 Vue Directive 是和 DOM 元素的創建、銷毀綁定的。 Vue Directive 的生命週期方法能讓我們更優雅的在合適的時機進行 DOM 的操作。而在 ViewModel 裡則沒有和 DOM 元素對應的方法。因為 Vue Directive 是屬於 View 層面的，所以DOM 操作應該被封裝在 Vue Directive 而不是出現在 Vue 實例中。
